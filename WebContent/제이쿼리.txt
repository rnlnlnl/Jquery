Ajax(Asynchronous JavaScript and XML)
	-> 비동기 방식의 자바 스크립트/xml
	
	xml: 데이터 타입
	 - 태그의 형태로 데이터를 저장하는 방식.
	 - 유연한 데이터 형태를 가지고 있음.
	 - HTML과 동일하게 DOM메서드를 사용가능하다.
	 - 다른 프로그램들과의 데이터 통신에서 많이 사용.
	 - 데이터 보다 태그의 크기가 더 많이 필요하기 때문에 많은 처리시간이 발생한다. (nano초 차이다)쌓이고쌓이고하면 느림.
	 - 요청이 동일한 도메인에서 처리해야함.
	
	* 동기 방식의 데이터 처리
	 -> 서버의 데이터 처리, 페이지 이동시 처리 가능(화면의 깜빡임)
	 => 자료를 요청시 반드시 서버의 컴퓨터를 거쳐서 사용해야함.
	 그동안 사용자는 대기해야함
	 
	사용자(요청) -> 웹서버 -> 사용자(응답)
	
	
	* 비동기 방식의 데이터 처리
	 -> 자료를 요청시 서버컴퓨터에서 정보를 가져와서 사용
	 화면전환이 없음 (페이지 이동 O)

	
 	$(주소).ajax({속성});
	$.ajax(주소,{속성});

	$.ajax({
		url : "전송 페이지",
		type : "전송방식(get/post)",
		data : "전송할 데이터",
		dataType: "요청한 데이터 타입(html,xml,text,json)", 
		success: function(데이터(전송해준 페이지의)){
			전송 완료시 처리동작
		}
	)};


 * JSON 데이터 타입
  (JavaScript Object Notation)
  => 
  
  - 다른 도메인 주소에서 처리되는 요청 처리 가능
  - 데이터 표현이 매우 간결함(문자데이터 key,value)
  - 문법에 엄격한 데이터
  - js 코드를 실행가능한 데이터 => 악용 가능성이 높음.
    => 안전한 페이지에서 제공하는 JSON데이터만 사용 해야함.
    
    
  - { } : JSON 객체
  - [ ] : JSON 배열
  - key : value 기본 데이터 형태
   (속성) : (값)
  - 속성의 경우 ""를 사용 해서 표시, 값의 경우 데이터 형태에 따라서 표시변경 
  - 속성이 여러개인 경우 , 사용해서 표시
  
  - 정수형(2/8/16진수 X)
  - 실수(고정 소수점, 부동 소수점(지수로그))
  - Boolean
  - null
  - 문자 데이터 (""사용, 유티코드형태로 사용가능)
  - 배열
    [10,"AAA",true,{"a":3,"b":10},100]
    {"a":100,"b":[10,20,30] }
